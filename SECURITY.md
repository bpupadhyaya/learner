# Security Policy

## Supported Versions
As a fast-moving project, only the latest `main` branch is supported for security fixes.

## Reporting a Vulnerability
Please do not disclose security issues publicly.

1. Use GitHub Security Advisories (preferred):
   - Repository tab: `Security` -> `Advisories` -> `Report a vulnerability`.
2. If Security Advisories are unavailable, open a private maintainer discussion on GitHub.
3. Include:
   - Affected component(s)
   - Reproduction steps
   - Impact assessment
   - Suggested remediation (if known)
4. You will receive acknowledgment within 72 hours.

## Disclosure Process
- We validate and triage reports privately.
- We prepare and test a fix.
- We publish a patch and advisory notes.
- We credit reporters when requested.

## Out-of-Scope Reports
- Missing best-practice headers without exploit path.
- Denial-of-service requiring unrealistic resources.
- Social engineering without technical vulnerability.
